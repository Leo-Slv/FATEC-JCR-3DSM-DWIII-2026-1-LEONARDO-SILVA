Requisição HTTP (geralmente POST) → enviada para o Express.

2. ORM/ODM – Papel na aplicação
ORM (Object Relational Mapping) → usado com bancos relacionais (Postgres, MySQL).
ODM (Object Document Mapping) → usado com MongoDB (Mongoose).
Função principal: mapear tabelas/coleções em objetos da aplicação.

4. Mongoose – Campos obrigatórios e únicos
Para garantir unicidade e obrigatoriedade em um campo email:
const UserSchema = new mongoose.Schema({

5. Prisma ORM x Mongoose ODM
Prisma → ORM para bancos relacionais (Postgres, MySQL, SQLite, etc.)
Mongoose → ODM para MongoDB (NoSQL).
Ambos mapeiam dados → objetos, mas cada um é especializado em um tipo de banco.
